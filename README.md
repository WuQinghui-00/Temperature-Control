# STM32 TEC 温控系统

基于 STM32F407VET6 的 TEC 温度采集与闭环控制系统模拟验证平台。

---

## 项目简介

本项目实现了一套完整的 TEC 温控算法验证系统，涵盖底层驱动、信号采集、温度换算、增量式 PID 控制、继电反馈自整定、串口通信与上位机交互全链路。

**当前阶段说明：** 本系统目前处于**算法模拟验证阶段**，尚未接入真实 TEC 负载与 NTC 热敏电阻。验证方式为：使用 DAC8568 输出的电压信号**模拟 NTC 分压电压**，将该电压直接回环接入 ADS1256 差分输入端，由 ADC 读取后经温度模型反向换算为“模拟温度值”，以此验证温度采集、PID 控制与自动整定算法的正确性。DAC 输出 → ADC 采集 → 温度换算 → PID 计算 → DAC 输出调整，形成完整的数字闭环，逻辑与真实 TEC 温控系统完全一致。

**当前工程状态：** Keil 构建 **0 错误**，已生成 HEX 并完成硬件下载验证，所有功能模块均测试通过。


## 硬件平台

| 组件 | 型号 | 规格说明 | 当前使用方式 |
|---|---|---|---|
| 主控 | STM32F407VET6 | ARM Cortex-M4，168MHz | 运行控制算法与驱动 |
| ADC | ADS1256 | 24位 Δ-Σ ADC，8通道差分 | 采集 DAC 输出的模拟 NTC 分压信号 |
| DAC | DAC8568 | 16位，8通道，软件 SPI 时序 | 输出模拟 NTC 分压电压，回环至 ADC 输入端 |
| 通信接口 | USART1 | 115200-8-N-1，DMA 环形收发 | 上位机命令与数据回传 |

> **说明：** 当前平台以 DAC → ADC 回环方式模拟 NTC 电桥信号，验证温控算法逻辑。待后续接入真实 TEC 与 NTC 传感器后，DAC 输出驱动 TEC，ADC 采集真实 NTC 分压，形成物理闭环。


## 系统框架

### 信号流向（从上到下）

**上位机（串口终端）**

↓ 命令下发 / 数据回传（每500ms）
**UART DMA（RX/TX）**

↓
**STM32F407VET6 主循环（周期：200ms）**

主循环内部流程：

1. 串口接收 → 命令解析
2. 控制模式调度（手动模式 / 自动整定模式 / 闭环模式）
3. 根据模式执行：
   - 增量式 PID 控制，或
   - 继电反馈自动整定
4. 温度与电压双向换算（基于 NTC 电桥模型）
5. DAC8568 驱动输出模拟电压

↓
**DAC A 通道输出（0~5V，模拟 NTC 分压信号）**

↓ 物理回环（杜邦线）
**ADS1256 差分采集（AIN0 - AIN1）**

↓
**ADC 码值 → 换算为“模拟温度”**

↓
**回到 PID 控制器，形成闭环**


### 闭环控制七步流程

| 步骤 | 内容 |
|---|---|
| 第1步 | PID / 整定算法计算 DAC 输出码值 |
| 第2步 | DAC8568 输出电压（0~5V），模拟 NTC 分压 |
| 第3步 | 电压经杜邦线回环至 ADS1256 差分输入端 |
| 第4步 | ADS1256 采集电压，返回 24 位 ADC 码值 |
| 第5步 | ADC 码值换算为电压，再经 NTC 模型换算为“模拟温度” |
| 第6步 | 模拟温度与目标温度比较，PID 计算新的输出值 |
| 第7步 | 每 200ms 循环一次，形成完整数字闭环 |


> **闭环逻辑说明：** DAC 输出电压模拟的是 NTC 在某一温度下的分压值。ADC 采集该电压后，经 NTC 温度模型换算得到“模拟温度”。PID 将模拟温度与目标温度比较，计算新的 DAC 输出码值，形成“电压 → 温度 → 电压”的数字闭环。该逻辑与真实温控系统完全一致，只是物理量由“NTC 真实分压”替换为“DAC 模拟分压”。


## 主要功能模块

### 1. 信号采集（ADS1256 驱动）

- **工作模式**：差分采样（AIN0-AIN1），PGA=1，数据速率 50 SPS
- **读取方式**：软件 SPI 时序，读取 24 位有符号码值
- **滤波处理**：多次采样排序，去掉最大最小值后取平均，降低瞬时噪声
- **电压换算**：ADC原码 → 差分电压（结合 PGA 与参考电压计算）
- **当前采集对象**：DAC8568 输出的模拟 NTC 分压信号

### 2. NTC 温度换算

- **电桥结构**：10kΩ NTC + 10kΩ 分压电阻串联，参考电压 5V
- **正向换算（真实场景）**：节点电压 → NTC电阻 → 温度（基于 B=3950 模型）
- **反向换算（模拟验证）**：温度 → NTC电阻 → 节点电压 → DAC 输出码值
- **有效性检查**：对电压超限、电阻超限、温度超限（-40°C ~ 150°C）进行保护
- **验证说明**：当前通过 DAC 输出电压模拟 NTC 分压，ADC 采集后经反向模型换算回温度，验证算法正确性

### 3. DAC 输出（DAC8568 驱动）

- **通信方式**：GPIO 模拟 SPI 时序（SCK、DIN、SYNC）
- **输出能力**：0 ~ 5V（对应码值 0 ~ 65535）
- **通道分配**：
  - DAC A 通道 → 模拟 NTC 分压信号输出（连接至 ADS1256 AIN0-AIN1）
  - DAC C 通道 → 备用 / 参考通道
- **状态记录**：保存各通道最近输出码值，便于回传与调试
- **当前输出对象**：模拟 NTC 分压电压（非真实 TEC 驱动）

### 4. 增量式 PID 闭环控制

**控制算法：**

Δu(k) = Kp·[e(k)-e(k-1)] + Ki·e(k) + Kd·[e(k)-2e(k-1)+e(k-2)]

**工程优化：**
- 小误差区域（|e| < 0.5°C）进行误差平滑，降低稳态抖动
- 分段增量衰减，防止积分饱和
- PID 等效温度输出限幅（0 ~ 40°C），保护模拟负载

**验证方式：** 设定目标温度 → PID 计算 → DAC 输出模拟 NTC 分压 → ADC 采集 → 温度换算 → 误差计算 → PID 更新，形成完整数字闭环

### 5. 继电反馈 PID 自动整定

**整定原理：** 在目标温度附近交替输出高/低控制量（通过 DAC 调整模拟 NTC 电压），使 ADC 换算出的“模拟温度”产生等幅振荡，记录振荡周期与幅值，计算出临界增益 Ku 与临界周期 Tu，代入 PID 参数公式。

**整定流程：**
1. 判断当前模拟温度是否接近目标温度（±2°C 以内）
2. 交替输出高低控制量（通过 DAC 改变模拟 NTC 电压）
3. 检测模拟温度越过迟滞上下限，记录极值点与周期
4. 连续 3 个周期稳定后计算平均幅值与周期
5. 计算 Ku 与 Tu → 采用 **Tyreus-Luyben 规则**计算 Kp、Ki、Kd
6. 将连续域参数转换为离散增量式参数（控制周期 200ms）
7. 自动写入 PID 参数，切换至闭环控制

**安全机制：**
- 温度安全限制（目标温度±5°C 范围保护）
- 总超时保护（60秒未完成整定则自动退出）
- 半周期超时保护（10秒未越过迟滞则重新开始）
- 振荡幅值过低检测（防止误判）
- 失败自动恢复（可重新触发整定）
- 支持手动终止与进度查询

### 6. 串口通信与上位机交互

- **物理层**：USART1，波特率 115200，8 位数据，1 位停止位，无校验
- **接收方式**：DMA Circular 模式 + USART IDLE 中断
- **发送方式**：DMA Normal 模式
- **命令解析**：以 `\r\n` 结尾，支持以下命令：

| 命令 | 格式 | 说明 |
|---|---|---|
| 设定目标温度 | `SETTEMPERATURE=25.0` | 范围：-40 ~ 150°C |
| 设定工作电流 | `SETCIRCUT=1.5` | 范围：0 ~ 3.0A |
| 手动 DAC 输出 | `SETDAC0=2048` | 通道0码值，范围：0 ~ 65535 |
| 手动 DAC 输出 | `SETDAC2=1024` | 通道2码值，范围：0 ~ 65535 |

- **数据回传**（每 500ms）：
  - 当前模拟温度值（°C）
  - ADC 差分电压（V）——即 DAC 输出的模拟 NTC 分压值
  - DAC 输出电压（V）
  - ADC 原始码值
  - 当前控制模式

## 快速开始

### 环境准备

| 工具 | 说明 |
|---|---|
| Keil MDK 5 | ARM 开发环境（需安装 STM32F4 器件支持包） |
| STM32CubeMX | 用于查看或重新生成工程配置 |
| 串口调试工具 | SecureCRT / Putty / SSCOM 等，用于上位机通信 |
| ST-Link / J-Link | 下载调试器 |
| **杜邦线（信号回环）** | **将 DAC8568 A 通道输出引脚连接至 ADS1256 AIN0 与 AIN1 输入端** |

### 硬件连接

**当前模拟验证阶段的连接方式：**

| 外设 | 引脚 | 功能 | 连接目标 |
|---|---|---|---|
| DAC8568 | DAC A 通道输出 | 输出模拟 NTC 分压信号 | → 接至 ADS1256 AIN0 |
| DAC8568 | GND | 参考地 | → 接至 ADS1256 AIN1 |
| ADS1256 | AIN0、AIN1 | 差分采集 DAC 输出信号 | ← 来自 DAC8568 输出 |
| USART1 | PA9(TX)、PA10(RX) | 串口通信 | → 接 USB-TTL 模块 |

> **信号路径：** DAC 输出电压（模拟 NTC 分压）→ 物理回环 → ADC 采集 → 温度换算 → 闭环控制。

### 编译与下载

1. 使用 Keil 打开 `MDK-ARM/TempCubeMX.uvprojx`
2. 编译（F7），确认 0 错误
3. 连接下载器，点击 Load（F8）下载至 STM32F407VET6

### 串口调试

- 波特率：115200
- 数据位：8
- 停止位：1
- 无校验
- 发送命令时以 `\r\n` 结尾（通常串口工具按回车键即可）
